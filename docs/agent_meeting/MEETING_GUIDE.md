# 에이전트 회의 가이드 (2026-03-05)

> 이 폴더는 여러 AI 에이전트가 사용자가 제시한 안건을 논의하는 공간입니다.

---

## 파일 유형 2가지

| 유형 | 파일명 | 작성자 | 수정 규칙 |
|------|--------|--------|----------|
| **안건** | `YYYY-MM-DD_HHMM_안건_<주제>.md` | 사용자가 지정한 에이전트 1명 | 회의 진행 중 수정 금지, **회의 종료 시 최종 결정 반영 수정 허용** |
| **회의록** | `YYYY-MM-DD_HHMM_회의록_<주제>.md` | 안건 미작성 에이전트가 먼저 생성 | 번갈아가며 섹션 추가 |

---

## 회의 진행 흐름

```
① 사용자 → 안건 주제 + 작성 에이전트 지정
      ↓
② 지정된 에이전트 → 안건 파일 작성 (회의 중 수정 금지)
      ↓
③ 반대 에이전트 → 회의록 파일 생성 + 첫 번째 응답 작성
      ↓
④ 안건 작성 에이전트 → 회의록에 반응/추가
      ↓
⑤ ③~④ 반복 (번갈아가며)
      ↓
⑥ 사용자 → 최종 결정
      ↓
⑦ 안건 파일 최종 업데이트(결정 반영) + 완료 폴더 아카이브
```

**예시**:
- 사용자가 클로드에게 안건 요청 → 클로드가 안건 작성 → 코덱스가 회의록 첫 응답 → 클로드 반응 → 코덱스 반응 → ...
- 사용자가 코덱스에게 안건 요청 → 코덱스가 안건 작성 → 클로드가 회의록 첫 응답 → 코덱스 반응 → 클로드 반응 → ...

## 안건 파일 형식

```markdown
# 안건 — <주제> (YYYY-MM-DD HH:MM)

> 작성: <에이전트명>
> 요청자: 사용자

## 배경
(왜 이 주제를 논의하는가)

## 핵심 질문
(이번 논의에서 결론을 내야 할 질문 목록)
1.
2.

## 현황 분석
(관련 데이터, 기술 검토, 근거)

## 작성자 초안 의견
(안건 작성 에이전트의 초기 입장)

## 사용자 결정 필요 사항
- [ ]
```

## 핵심 규칙

1. **안건은 사용자가 정한다** — 에이전트가 임의로 안건 생성 금지
0. **타임스탬프는 반드시 시스템 시각 기준** — 작성 직전 `date +"%H:%M"` 명령으로 실제 시각 확인 후 기입. 임의 추정 금지
2. **안건 파일은 회의 중 수정 금지** — 단, 회의 종료 후 사용자 결정 반영 업데이트는 허용
3. **회의록은 반드시 반대 에이전트가 먼저 시작** — 안건 작성자가 먼저 응답 금지
4. **회의록에서 번갈아 작성** — 같은 에이전트가 연속 2회 작성 금지
5. **별도 질문/확인 파일 생성 금지** — 회의록 안에서 섹션으로 처리
6. **최종 결정은 사용자** — 에이전트는 의견 제시만
7. **미팅 가이드에 현재 미팅상황 표 작성 금지** — 상태 관리는 회의록/안건 파일에서만 관리

---

## 회의 종료/아카이브 규칙

- 회의 종결 시 안건/회의록을 함께 아카이브한다.
- 아카이브 경로:
```
docs/agent_meeting/completed/YYYY-MM-DD_HHMM_<주제>/
```
- 처리 순서:
1. 안건 파일에 사용자 최종 결정을 반영 (허용된 종료 수정)
2. 안건 파일 + 회의록 파일을 완료 폴더로 **복사**
3. 루트 `docs/agent_meeting/`의 해당 안건/회의록 원본은 완료 폴더로 **이동**하여 진행 중 파일과 분리
